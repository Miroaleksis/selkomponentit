Slide Carousel

A slide carousel cycles through a sequence of content panels, displaying one at a time. Users navigate between slides using previous and next buttons or indicator dots.

Slide carousels can be used when one area of the page must showcase several related but independent items, such as featured products, testimonials, or images of a single object from different angles. They are less suitable for primary content, since slides that are not currently visible are easily overlooked.

Accessibility Features

= Required = Recommended
  • The carousel is wrapped in role="region" with aria-roledescription="carousel" and an aria-label, so screen readers announce its purpose
  • Each slide uses role="group" with aria-roledescription="slide" and aria-label="Slide N of M" to communicate its position
  • Inactive slides use aria-hidden="true" and inert, so they are removed from both the accessibility tree and the focus order
  • The slides container uses aria-live="polite" so screen readers announce each slide change, whether triggered by the user or automatically
  • Auto-rotating carousels should provide a pause control, since automatic motion can interrupt the user and is difficult to follow with a screen reader

Static Carousel

A static carousel does not advance automatically. Users navigate between slides using the previous and next buttons or the indicator dots. Indicator buttons use aria-label="Slide N" to announce their purpose, and the active indicator is marked with aria-current="true".

Auto-rotating Carousel

An auto-rotating carousel advances slides automatically at a set interval. A play/pause control lets users stop and resume the rotation. The slides container uses aria-live="polite" so screen readers announce each slide change, whether triggered by the user or automatically.
