Carousel

A carousel cycles through a sequence of content panels, displaying one at a time. Users navigate between slides using previous and next buttons or indicator dots.

Carousels can be used when one area of the page must showcase several related but independent items, such as featured products, testimonials, or images of a single object from different angles. They are less suitable for primary content, since slides that are not currently visible are easily overlooked.

Accessibility Features

= Required = Recommended
  • The carousel is wrapped in role="region" with aria-roledescription="carousel" and aria-label to communicate its purpose
  • Each slide uses aria-roledescription="slide" to communicate its purpose
  • Carousel controls have descriptive accessible names via aria-label
  • A play/pause control lets users stop and resume auto-rotation, since automatic motion can disrupt the user
  • Slides can be navigated using previous/next buttons or tabs. If tabs are used, appropriate attributes and keyboard interaction patterns are required. See the Tabs page for details
  • Inactive slides use inert, so they are removed from the accessibility tree and focus order, and excluded from pointer events
  • The slides container uses aria-live="off" by default. On manual navigation it is temporarily set to aria-live="polite" so screen readers announce the new slide
