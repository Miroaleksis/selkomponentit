Pagination

Pagination allows users to navigate between pages of content, one page at a time.

Pagination can be used when a long list or set of results is divided into discrete pages, especially when users benefit from a clear sense of how much content is available and the ability to jump to specific pages. Infinite scroll or "load more" patterns may be more appropriate when the content is open-ended or browsed casually.

Accessibility Features

= Required = Recommended
  • Wrapped in a <nav> landmark with aria-label="Pagination" to identify the navigation region
  • aria-current="page" marks the currently active page for screen readers
  • The disabled attribute on the Previous and Next buttons prevents interaction when no further navigation is possible
  • Use <button> when pagination updates content dynamically without a page reload. Use <a> when each page has its own URL
  • Uses an ordered list (<ol>) to communicate the total number of pages to screen readers
  • A role="status" element above the pagination announces the current page to screen readers when updated dynamically
