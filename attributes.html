Attributes

This page provides a reference of elements and attributes used in this library. Both native HTML elements and attributes and their corresponding ARIA attributes are listed side by side.

Native HTML elements and attributes come with built-in behaviors and semantics. ARIA (Accessible Rich Internet Applications) attributes supplement HTML when no native solution fits, when a custom component needs additional semantics, or when the built-in behaviors of a native element are unwanted. Always prefer the native element when possible.

This page covers the elements and attributes commonly needed when building accessible components. It is not an exhaustive reference, multiple elements and attributes beyond what is listed here exist.

Roles define the purpose of an element. Native elements come with built-in roles and behaviours and should be used when possible. ARIA roles only communicate semantics – behaviours must be implemented separately.
Native Role Description
<a> link A navigable hyperlink.
— menu A list of choices or commands, typically opened from a button.
— menuitem An individual option or command within a menu.
<nav> navigation A set of navigation links.
— tab An interactive element in a tablist that controls the visibility of its associated tabpanel.
— tablist A container for a set of tab elements.
— tabpanel A content panel associated with a tab element.
<button> button An interactive element that triggers an action.
<input type="checkbox"> checkbox A control with three possible states: checked, unchecked, or mixed.
— combobox An input control that displays a dropdown list of options.
<select> / <option> listbox / option A list of options from which the user can select one or more items. Each <option> represents a selectable item within the list.
<input type="radio"> radio A single option within a radiogroup.
<input type="search"> searchbox A text input intended for entering search queries.
<input type="range"> slider An input for selecting a value within a range by dragging a thumb.
<input type="number"> spinbutton A numeric input that allows incrementing and decrementing a value with arrow buttons.
— switch A toggle button representing an on/off state, similar to a checkbox but with binary semantics.
<input type="color"> — A color picker that lets users select a color value.
<input type="date"> — A date picker that lets users select a date.
<input type="file"> — A file upload control that opens the system file dialog.
<label> — Associates a visible text label with a form control. Linked via matching for and id attributes or by nesting the control inside the <label>. Note that multiple inputs (or other interactive elements) must not be nested inside a single <label>.
<header> banner A header region for the page or its nearest sectioning ancestor, such as an <article>.
<td> cell A data cell within a table row.
<th> columnheader A header cell for a column in a table.
<aside> complementary Supplementary content that supports but is not essential to the main content.
<footer> contentinfo A footer region region for the page or its nearest sectioning ancestor, such as an <article>.
<fieldset> / <legend> group A container that groups related form controls. The <legend> provides a caption for the group.
<ul>, <ol> list A list of items.
<li> listitem A single item within a list.
<main> main The primary content of the page. Each page should have only one main region.
— presentation / none Removes an element's native semantics from the accessibility tree. Used to suppress redundant roles on decorative or wrapper elements.
<progress> progressbar An element that displays the progress of a task, either determinate or indeterminate.
<section> region A named section of a page. Only exposed as a landmark when it has an accessible name.
<tr> row A row of cells within a table or grid.
<thead>, <tbody>, <tfoot> rowgroup A group of rows within a table.
<th scope="row"> rowheader A header cell for a row in a table.
<table> table A grid of data arranged in rows and columns.
<details> / <summary> — A native accordion widget that expands and collapses content. The <summary> acts as the visible toggle.
— alert A live region for an important, time-sensitive message that is announced immediately by screen readers. Behaves as aria-live="assertive" and aria-atomic="true" by default.
— alertdialog A dialog containing an urgent message that requires immediate user attention. Unlike dialog, screen readers interrupt the current flow and announce the content immediately.
<dialog> dialog A window overlaid on the page that requires user interaction before continuing.
<output> status A live region for polite status updates that do not require immediate attention. Behaves as aria-live="polite" and aria-atomic="true" by default.
— tooltip A short descriptive popup that appears on hover, focus or press of its associated element.
Attributes that only accept a fixed set of values. States are dynamic attributes that change with user interaction. Properties are static attributes that rarely change, set by the author to describe an element.
Native ARIA Values Description
— aria-atomic true / false When set to true, instructs assistive technologies to announce the entire live region when any part of it changes, rather than only the changed portion.
— aria-autocomplete none / list / inline / both Indicates to assistive technologies how text completion is provided in a combobox. list means a dropdown list of suggestions is displayed.
checked aria-checked true / false / mixed Indicates the checked state of a checkbox, radio button or switch.
— aria-current page / step / date / true Indicates the current item within a set, such as the active page in a navigation or the current step in a stepper.
disabled aria-disabled true / false Indicates that an interactive element is disabled. The native disabled attribute also prevents interaction rather than only communicating it.
— aria-expanded true / false Indicates whether a collapsible element such as an accordion is open or closed.
— aria-haspopup true / listbox / menu / dialog Indicates that interacting with the element will open a popup such as a listbox, menu or dialog.
hidden aria-hidden true / false Removes an element from the accessibility tree so assistive technologies ignore it. The native hidden attribute also removes the element visually and from the tab order. When using aria-hidden on interactive elements, add tabindex="-1" to also remove them from the tab order.
— aria-invalid true / false / grammar / spelling Indicates that the value of a form field does not meet the expected format or requirements. Required when using aria-errormessage.
— aria-live polite / assertive / off Defines a live region whose content changes are announced by screen readers. Common values are polite (waits for user to be idle) and assertive (announces immediately).
— aria-pressed true / false / mixed Indicates the pressed state of a toggle button.
required aria-required true / false Indicates that the field must be filled in before the form can be submitted. The native required attribute also prevents form submission and triggers built-in browser validation.
— aria-selected true / false Indicates the currently selected item within a set such as a tab or option.
autocomplete — name / email / off / … Hints to the browser what type of data the field expects, enabling autofill. Common values include name, email, tel and off. See all autocomplete values on MDN.
inert — — Removes an element and all its descendants from focus order, pointer events and the accessibility tree.
tabindex — 0 / -1 Controls whether an element is reachable via keyboard navigation. A value of 0 adds the element to the natural tab order. A value of -1 makes it focusable programmatically but removes it from the tab order. Positive values are not recommended.
Attributes that provide accessible names and descriptions as text strings directly in the markup.
Attribute Description
aria-description Provides an accessible description directly as a text string. Use when no visible description is present.

Not yet supported in all browsers and assistive technologies. Use aria-describedby for broader compatibility.
aria-label Provides an accessible name directly as a text string. Use when no visible label is present.
aria-roledescription Provides a human-readable description of the element's role, overriding the default role name announced by assistive technologies.
aria-valuetext Provides a human-readable text alternative for the current numeric value of a range widget, announced by screen readers instead of the raw number.
Attributes that reference the id of another element to create a relationship between elements.
Attribute Description
aria-activedescendant References the id of the currently active descendant element within a composite widget such as a listbox or combobox, without moving DOM focus away from the container.
aria-controls References the id of the element that this element controls, such as a button that opens a panel.
aria-describedby References the id of an element that provides additional descriptive information about the current element.
aria-details References the id of an element that provides additional information about the current element, beyond what aria-describedby offers.
aria-errormessage References the id of an element that contains an error message for the current element. Only relevant when aria-invalid is set.
aria-labelledby References the id of another element whose text serves as the accessible name for the current element.
aria-owns References elements that are logically owned by the current element but are not its DOM children.
