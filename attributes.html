Attributes

A reference of ARIA roles and attributes used in this library. ARIA (Accessible Rich Internet Applications) supplements HTML to communicate element purpose, state and relationships to assistive technologies.

This page covers the roles and attributes most commonly needed when building accessible components. It is not an exhaustive reference — the full ARIA specification includes many additional roles and attributes beyond what is listed here.

Roles

Native Role When to use
<a> link A navigable hyperlink. Use the native element instead of the role where possible.
— menu A list of choices or commands, typically opened from a button.
— menuitem An individual option or command within a menu.
<nav> navigation A set of navigation links. Use the native element instead of the role where possible.
— tab An interactive element in a tablist that controls the visibility of its associated tabpanel.
— tablist A container for a set of tab elements.
— tabpanel A content panel associated with a tab element.
<button> button An interactive element that triggers an action. Use the native element instead of the role where possible.
<input type="checkbox"> checkbox A control with three possible states: checked, unchecked, or mixed. Use the native element instead of the role where possible.
— combobox A text input paired with a popup listbox of suggested values.
<select> listbox A list of options from which the user can select one or more items.
<option> option A selectable item within a listbox or combobox popup.
<input type="radio"> radio A single option within a radiogroup. Use the native element instead of the role where possible.
<fieldset> radiogroup A group of radio buttons where only one can be selected at a time.
<input type="range"> slider An input for selecting a value within a range by dragging a thumb. Use the native element instead of the role where possible.
<input type="number"> spinbutton A numeric input that allows incrementing and decrementing a value with arrow buttons.
— switch A toggle button representing an on/off state, similar to a checkbox but with binary semantics.
<input type="color"> — A color picker that lets users select a color value. Has no ARIA equivalent.
<input type="date"> — A date picker that lets users select a date. Has no ARIA equivalent.
<input type="file"> — A file upload control that opens the system file dialog. Has no ARIA equivalent.
<header> banner Site-level header region. Use the native element instead of the role where possible.
<td> cell A data cell within a table row.
<th> columnheader A header cell for a column in a table.
<aside> complementary Supplementary content that supports but is not essential to the main content.
<footer> contentinfo Site-level footer region containing copyright and secondary links.
<fieldset> group A container that groups related elements, such as a set of radio buttons.
<ul>, <ol> list A list of items. Use the native element instead of the role where possible.
<li> listitem A single item within a list. Use the native element instead of the role where possible.
<main> main The primary content of the page. Each page should have only one main region.
— presentation / none Removes an element's native semantics from the accessibility tree. Used to suppress redundant roles on decorative or wrapper elements.
<progress> progressbar An element that displays the progress of a task, either determinate or indeterminate.
<section> (with accessible name) region A named section of a page. Only exposed as a landmark when it has an accessible name.
<tr> row A row of cells within a table or grid.
<thead>, <tbody>, <tfoot> rowgroup A group of rows within a table.
<th scope="row"> rowheader A header cell for a row in a table.
<table> table A grid of data arranged in rows and columns. Use the native element instead of the role where possible.
<details> / <summary> — A native disclosure widget that shows and hides content without JavaScript. The <summary> acts as the visible toggle. Has no ARIA equivalent.
— alert An important, time-sensitive message that is announced immediately by screen readers.
— alertdialog A dialog containing an urgent message that requires immediate user attention. Unlike dialog, screen readers interrupt the current flow and announce the content immediately.
— dialog A window overlaid on the page that requires user interaction before continuing.
— status A live region for polite status updates that do not require immediate attention.
— tooltip A short descriptive popup that appears on hover or focus of its associated element.

States

Native ARIA Values Description
checked aria-checked true / false / mixed Indicates the checked state of a checkbox, radio button or switch.
— aria-current page / step / date / true Indicates the current item within a set, such as the active page in a navigation or the current step in a stepper.
disabled aria-disabled true / false Indicates that an element is visible but not currently interactive. The native disabled attribute is stronger — it actually prevents interaction rather than only communicating it.
— aria-expanded true / false Indicates whether a collapsible element such as an accordion panel or dropdown is open or closed.
hidden aria-hidden true / false Removes an element from the accessibility tree so assistive technologies ignore it. The native hidden attribute also removes the element visually.
:invalid aria-invalid true / false / grammar / spelling Indicates that the value of a form field does not meet the expected format or requirements. Required when using aria-errormessage.
— aria-pressed true / false / mixed Indicates the pressed state of a toggle button.
— aria-selected true / false Indicates the currently selected item within a set such as a tab or option.

Properties

Native ARIA Description
— aria-atomic When set to true, instructs assistive technologies to announce the entire live region when any part of it changes, rather than only the changed portion.
— aria-describedby References the id of an element that provides additional descriptive information about the current element.
— aria-description Provides an accessible description directly as a text string. Use when no visible description is present.

Not yet supported in all browsers and assistive technologies. Use aria-describedby for broader compatibility.
— aria-haspopup Indicates that interacting with the element will open a popup such as a listbox, menu or dialog.
— aria-label Provides an accessible name directly as a text string. Use when no visible label is present.
— aria-labelledby References the id of another element whose text serves as the accessible name for the current element.
— aria-live Defines a live region whose content changes are announced by screen readers. Common values are polite (waits for user to be idle) and assertive (announces immediately).
required aria-required Indicates that the field must be filled in before the form can be submitted. The native required attribute is stronger — it also prevents form submission and triggers built-in browser validation.
— aria-roledescription Provides a human-readable description of the element's role, overriding the default role name announced by assistive technologies.
— aria-valuetext Provides a human-readable text alternative for the current numeric value of a range widget, announced by screen readers instead of the raw number.

Relationships

Attribute Description
aria-activedescendant References the id of the currently active descendant element within a composite widget such as a listbox or combobox, without moving DOM focus away from the container.
aria-controls References the id of the element that this element controls, such as a button that opens a panel.
aria-details References the id of an element that provides additional information about the current element, beyond what aria-describedby offers.
aria-errormessage References the id of an element that contains an error message for the current element. Only relevant when aria-invalid is set.
aria-owns References elements that are logically owned by the current element but are not its DOM children.
