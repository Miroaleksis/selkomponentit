Stepper

A stepper guides users through a multi-step process by displaying all steps and indicating progress.

Steppers can be used when a task is divided into a fixed sequence of distinct steps that benefit from being shown as a whole, such as a multi-step form, checkout flow, or onboarding wizard. They are less suitable for short tasks of one or two steps, or branching processes where the next step depends on previous answers.

Accessibility Features

= Required = Recommended
  • Wrapped in a <nav> landmark with aria-label to identify the progress region
  • Uses an ordered list (<ol>) to convey the sequence and total number of steps to screen readers
  • The current step's button is marked with aria-current="step"
  • Upcoming step buttons use the disabled attribute, so screen readers and keyboard users know they cannot yet be navigated to
  • Use <button> when the stepper navigates dynamically without a page reload. Use <a> when each step has its own URL
  • An aria-live="polite" region announces the current step's label when it changes, so screen reader users hear the update without losing focus
  • Step state (completed, current, upcoming) is conveyed through icon, font weight and disabled state, not by colour alone
