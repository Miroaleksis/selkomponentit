Table

A table organises data into rows and columns, making it easy to scan and compare information. Tables should only be used for tabular data — not for layout purposes.

Interactive tables extend the basic structure with sorting and expandable rows, allowing users to control how data is presented and revealed.

Accessibility Features

= Required = Recommended
  • <caption> provides a visible title for the table and is announced by screen readers before the table content
  • scope="col" on <th> elements identifies them as column headers
  • aria-sort="ascending|descending|none" on a sortable column header communicates the current sort direction to screen readers
  • Sort controls use a <button> inside <th> so they are reachable by keyboard
  • aria-expanded and aria-controls on expand buttons communicate the state and target of the detail row
  • Sort direction icons are decorative and hidden from assistive technology — the aria-sort attribute conveys the same information
