Landmarks

Landmark regions are structural areas of a page defined by semantic HTML elements. Screen readers expose landmarks so users can jump directly to any section without navigating through all content first.

Native HTML elements such as <header>, <nav>, <main>, <aside> and <footer> carry implicit landmark roles and should always be preferred over adding role attributes to generic elements.

Accessibility Features

= Required = Recommended
  • Each page must have exactly one <main> element
  • When the same landmark type appears more than once — for example two <nav> elements — each must have a unique accessible name via aria-label or aria-labelledby
  • A <section> element only becomes a landmark region when it has an accessible name; without one it is treated as a generic container
  • All visible content should be contained within a landmark region so nothing is left outside the landmark structure
  • Use as few landmarks as needed to describe the page structure — excessive landmarks reduce their usefulness for screen reader navigation
