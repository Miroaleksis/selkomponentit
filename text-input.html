Text Input

A text input is a basic form control that allows users to enter freeform text. It is the most commonly used input field.

Accessibility Features

= Required = Recommended
  • Every input must have a visible <label> associated via matching for and id attributes or nesting. Note that multiple inputs (or other interactive elements) must not be nested inside a single <label>
  • The help text, error messages and prefixes/suffixes must be associated with the input using the aria-describedby attribute
  • Use autocomplete to help users fill in common fields such as name, email and address
  • Placeholder text must not be used as a substitute for a label as it is not permanent and disappears on input
  • The required state should be indicated clearly with text. If a symbol like an asterisk (*) is used, it should be explained at the top of the form. required attribute helps to communicate the information programmatically

Single-line Text

The native <input type="text"> element should always be used for single-line text, as it provides built-in keyboard support, screen reader announcements and mobile keyboard optimisation.

Textarea

A <textarea> allows users to enter multiline text. The maxlength attribute limits the number of characters and a visible counter keeps users informed of the remaining characters.
