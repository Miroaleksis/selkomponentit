Text Input

A text input is a single-line form control that allows users to enter freeform text, such as a name, email address or search query.

The native <input> element should always be used, as it provides built-in keyboard support, screen reader announcements and mobile keyboard optimisation. The type attribute should reflect the expected input — for example type="email" or type="search" — to trigger the appropriate virtual keyboard on mobile devices.

Accessibility Features

= Required = Recommended
  • Every input must have a visible <label> associated via nesting or matching for and id attributes
  • The help texts, error messages and prefixes/suffixes must be associated with the input using the aria-describedby attribute
  • Use autocomplete to help users fill in common fields such as name, email and address
  • Placeholder text must not be used as a substitute for a label. It disappears on input and has low contrast by default
  • The required state should be indicated clearly with text. If a symbol like an asterisk (*) is used, it should be explained at the top of the form
