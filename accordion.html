Accordion

An accordion allows users to expand and collapse sections of content as needed.

Accordions can be used when several related sections share the same context and reducing visual clutter is helpful. They are less suitable for content that users need to compare side by side, since toggling adds friction.

The recommended implementation is the native <details> and <summary> elements, which provide expand/collapse behaviour and the necessary accessibility features without JavaScript.

Accessibility Features

= Required = Recommended
  • Expands and collapses with Space and Enter
  • The expanded/collapsed state must be communicated programmatically so screen readers can announce whether a panel is expanded or collapsed

Details Element

The native <details> and <summary> elements handle expand/collapse behaviour and communicate state to screen readers automatically, without JavaScript or additional ARIA attributes.

Custom Accordion

A custom accordion requires aria-expanded on the trigger button to communicate the expanded or collapsed state to screen readers. The content panel can use role="region" with aria-labelledby pointing to the trigger to expose it as a named landmark. JavaScript is needed to toggle the attribute when the user activates the trigger.
