Alert

An alert is a message that communicates important feedback to the user in response to an action or event. There are two variants: a status message for non-critical updates and an alert message for errors or urgent information.

Alerts use ARIA live regions to announce their content to screen readers automatically when they appear in the page, without requiring the user to navigate to them.

Accessibility Features

= Required = Recommended
  • role="status" announces non-critical updates politely. Screen readers wait until the user is idle before reading the message
  • role="alert" announces urgent messages assertively. Screen readers interrupt the current task to read the message immediately
  • The live region container is present in the page before the message appears. Screen readers only announce changes within the region, so the container can only be hidden visually, not removed from the DOM
  • The alert message conveys clearly the nature of the detected issues
  • If the form is long, the alert message can also include list of links to all invalid fields
  • When submitting the form fails, focus should be set to the alert message or the first invalid field

Example

Leave the form fields empty and click the submit button to see the alert message. Fill in all required fields to see the status message.
