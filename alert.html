Alert

An alert is a message that communicates important feedback to the user in response to an action or event. There are two variants: a status message for non-critical updates and an alert message for errors or urgent information.

Alerts use ARIA live regions to announce their content to screen readers automatically when they appear in the page, without requiring the user to navigate to them.

Browser provides native input error messages when input fields are wrapped inside a <form> element (See the Form page for more information).

Accessibility Features

= Required = Recommended
  • role="status" announces non-critical updates politely. Screen readers wait until the user is idle before reading the message
  • role="alert" announces urgent messages assertively. Screen readers interrupt the current task to read the message immediately
  • The alert container must be present in the page before the message appears. Screen readers only announce changes within the container, so it must only be hidden visually, not removed from the DOM
  • The alert message must convey clearly the nature of the detected issues
  • If there are multiple invalid fields, the alert message may also include a list of links to each field
  • When an input error causes the alert message, the invalid field should be marked with aria-invalid="true" to communicate its state to screen readers. The field can also be associated with its error message using either aria-errormessage or aria-describedby
  • Avoid combining focus management with live region announcements unless focus moves to the live region itself. Screen readers prioritize focus changes and may not announce the live region. Generally, use one or the other: either move focus to the first incorrectly filled field (in this case, the field must have an associated error message), or rely on the live region to announce the error.
