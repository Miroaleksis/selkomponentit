Select-Only

A select-only combobox allows users to select a single option from a dropdown list. It can be used whenever users must choose from a known, limited set of predefined values. It does not allow freeform text entry. The recommended implementation is to use the native <select> element as it has built-in accessibility.

Accessibility Features

= Required = Recommended
  • A combobox expands and collapses with Space and Enter when the trigger element is focused
  • The expanded/collapsed state must be communicated programmatically to screen readers
  • Options are navigable with ↑ and ↓ arrow keys and selected with Space and Enter
  • Esc collapses the dropdown and returns focus to the trigger
  • Moving focus away from the dropdown with Tab collapses the dropdown automatically
  • Home and End move focus to the first and last option

Select Element

The native <select> element provides keyboard support, screen reader announcements and platform-native rendering for free. Its styling is limited across browsers, which is why some users may prefer a custom option.

Custom Select-Only

A custom implementation may be necessary when the visual design cannot be achieved with the native <select> element. It requires explicit ARIA to communicate widget semantics: role="combobox" on the trigger element identifies the widget, and aria-expanded communicates the expanded or collapsed state to screen readers.
