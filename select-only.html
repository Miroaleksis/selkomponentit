Select-Only

A select-only combobox allows users to select a single option from a dropdown list. It can be used whenever users must choose from a known, limited set of predefined values. It does not allow freeform text entry.

Accessibility Features

= Required = Recommended
  • The role and state of the trigger element and options must be communicated programmatically to screen readers
  • A combobox expands and collapses with Space and Enter when the trigger element is focused
  • Options are navigable with ↑ and ↓ arrow keys and selected with Space and Enter
  • Esc collapses the dropdown and returns focus to the trigger
  • Moving focus away from the dropdown with Tab collapses the dropdown automatically
  • Home and End move focus to the first and last option

Select Element

The native <select> element handles keyboard support, screen reader announcements and dropdown behaviour automatically. Using it is recommended when possible. Its styling is limited across browsers however, which is why some may prefer a custom implementation.

Custom Select-Only

A custom implementation may be necessary when the desired visual design cannot be achieved with the native <select> element. It requires explicit attributes to communicate dropdown semantics:

  • role="combobox" identifies the trigger element and aria-expanded communicates its expanded or collapsed state
  • Each option has role="option" and aria-selected to communicate its selection state
  • aria-controls should point to the element the trigger controls
