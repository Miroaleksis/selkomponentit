Combobox with Search

A combobox with search includes a text input that filters the dropdown list of options as the user types. It is suited for longer lists where browsing all options one at a time would be impractical. It should not be used when the list of options is short or limited, as the search feature adds unnecessary complexity and is susceptible to accessibility issues.

The combobox with search supports single selection and multiselection.

Accessibility Features

= Required = Recommended
  • A combobox with search has a similar structure and keyboard behaviour as the select-only combobox and the multiselect combobox. However, the implementation differs in a few ways:
  • input type="text" as the trigger element provides search function
  • aria-autocomplete="list" indicates that the displayed options are filtered based on the typed value
  • A role="status" element inside the listbox announces feedback to screen readers when no options match the search query

Multiselection

In a multiselection combobox, selected options are associated with the combobox via aria-describedby, since they cannot be nested within an input field. Pressing Backspace when the input is empty removes the last selected option and aria-live="polite" on the option container announces the change to screen readers
